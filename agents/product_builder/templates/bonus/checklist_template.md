# Checklist Template

Create phase-based action checklists that help readers track implementation and progress.

## Required Sections

### Introduction
- How to use the checklists
- Print recommendations
- Checking cadence

### Phase Checklists

For each major phase of the program:

**Phase Checklist Format:**
```
# [Phase Name] Checklist

## Goals for This Phase
[Brief description of what should be achieved]

## Daily Non-Negotiables
□ [Action item with specific detail]
□ [Action item]
□ [Action item]

## Weekly Targets
□ [Weekly goal]
□ [Weekly goal]

## By End of Phase
□ [Milestone 1]
□ [Milestone 2]
□ [Milestone 3]

## Markers of Progress
- [Observable sign of success]
- [Observable sign of success]

## Common Obstacles This Phase
- [Obstacle]: [Solution]
- [Obstacle]: [Solution]
```

Create checklists for:
- Foundation Phase (Days 1-30)
- Expansion Phase (Days 31-60)
- Integration Phase (Days 61-90)
- Maintenance (ongoing)

### Pillar-Specific Checklists
For each major pillar/area, create:
- Setup checklist (environment, supplies)
- Daily practice checklist
- Weekly review checklist

### Quick-Start Checklist
Condensed "do this first" list for someone just beginning

### Troubleshooting Checklists
When things go wrong:
- Reset checklist
- Recovery checklist
- Simplification checklist

## Format Requirements

- Use checkbox format: □ for items
- Group related items together
- Include success criteria where possible
- Keep items specific and actionable

## Quality Standards

- Items should be concrete (not "eat better" but "eat protein at breakfast")
- Clear completion criteria
- Organized by timeframe and category
- Target: 1200+ words minimum
